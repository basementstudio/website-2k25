## Codebase Patterns
- Fix scripts follow a consistent pattern: import dotenv/config, basehubClient, sanityWriteClient, then fetch from BaseHub → transform → createOrReplace in Sanity
- Use `fetchExistingIds(type)` to pre-load Sanity document IDs for reference validation before creating documents with references
- Use `slugify()` for deterministic document IDs: `{type}-{slug}`
- Image downloads from BaseHub are unreliable — always wrap `downloadAndUploadImage` in try/catch or use retry+timeout wrapper
- The `convertRichText` function calls `downloadAndUploadImage` internally for inline images, gallery images, and quote avatars — failures there propagate up and will crash the caller unless wrapped
- BaseHub returns polymorphic blocks in two formats (flat or nested under `on_<TypeName>`) — use `normalizeBlocks()` to handle both
- Sanity validation errors ("Bad Request - Validation failed") often come from image upload failures inside rich text conversion, not from the document structure itself
- Post categories, project categories should be created/refreshed before creating posts/projects that reference them
- BaseHub list types (BlockList) use `.items` accessor; single documents (BlockDocument) are accessed directly — check GraphQL schema to distinguish
- BaseHub rich text fields (like `content: Content_4`, `role: Role`) support `.plainText`, `.html`, `.json.content` accessors — use `.plainText` when Sanity schema expects plain text
- Sanity schema deployment requires user session auth (`sanity login`); the SANITY_API_TOKEN doesn't work — use MCP `deploy_schema` as fallback, converting defineType/defineField to plain object literals
- tsconfig path alias is `@/*` → `./src/*` — use `@/service/sanity` not `~/service/sanity`
- `next-sanity` v12+ provides `defineEnableDraftMode` from `next-sanity/draft-mode` for secure draft mode enable routes — handles Sanity Studio validation internally
- `next-sanity` v11+ subpath exports: `VisualEditing` from `next-sanity/visual-editing`, `defineLive` from `next-sanity/live` — NOT from root `next-sanity`
